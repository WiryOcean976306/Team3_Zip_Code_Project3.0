4) Scenario Matrix (Required Cases)
-----------------------------------
S1) Add requiring no block split
- Input: data/Length_Indicated_small_default.txt
- Options: blockSize 512, add file data/small_records_add_no_split.txt
- Expected: add log shows insert into existing block (no split)
- Evidence file: data/small_default_addition_log.txt

S2) Add requiring block split and avail-list use/update
- Input: data/Length_Indicated_small_merge_case.txt
- Options: blockSize 180, delete 10007, add 10012 in same run
- Design intent: deletion clears rightmost block to avail list, subsequent split reuses avail block
- Evidence files:
  - data/small_split_avail_deletion_log.txt
  - data/small_split_avail_addition_log.txt

S3) Delete requiring no block deletion or redistribution
- Input: data/Length_Indicated_small_default.txt
- Options: blockSize 512, delete file data/small_keys_delete_no_redistribution.txt
- Expected: delete success only, no redistribution/merge event
- Evidence file: data/small_delete_simple_deletion_log.txt

S4) Delete requiring redistribution with no merge
- Input: data/Length_Indicated_small_six_capacity.txt
- Options: blockSize 180, delete file data/small_keys_delete_redistribution.txt
- Expected: redistribution event logged
- Evidence file: data/small_redistribute_deletion_log.txt

S5) Delete requiring merge with logically rightmost block cleared and added to avail list
- Input: data/Length_Indicated_small_merge_case.txt
- Options: blockSize 180, delete file data/small_keys_delete_merge_rightmost.txt
- Expected: merge/clear event logged and block removed from active chain
- Evidence file: data/small_merge_deletion_log.txt
